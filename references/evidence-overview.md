# 文献与资源证据概述

本文件用于 Phase 3。它接收 Phase 2 已标准化的证据包，包括 PubMed evidence packet、arXiv methods packet、Web verification packet 和用户材料包，生成面向后续创新点判断的文献与资源概述。

Phase 3 只回答四个问题：

1. 当前材料覆盖了哪些研究主题。
2. 文献和资源分布如何。
3. 不同来源分别能支持什么，不能支持什么。
4. 进入创新点判断前还缺哪些证据。

Phase 3 不重新检索，不设计检索式，不补造缺失字段，不判断创新点，不生成假说，不做 Start/Narrow/Redesign/Stop 决策。

## 输入要求

Phase 3 只接收已经过 Phase 2 标准化的材料：

| 输入 | 来源 | 用途 |
|---|---|---|
| PubMed evidence packet | Phase 2 | 建立核心医学文献主干 |
| arXiv methods packet | Phase 2 | 补充 AI、统计、多组学、计算方法和预印本方法线索 |
| Web verification packet | Phase 2 | 核验数据集、临床试验、指南、数据库入口、官方页面和论文线索 |
| 用户材料包 | Phase 1/Phase 2 | 保留用户实际提供的论文、摘要、实验结果、笔记或导出材料 |

## 证据完整性规则

在生成概述前，先确认每类材料的状态、字段和边界。该规则统一处理“未获得”“不适用”“弱来源”和缺失字段，避免在后文重复说明。

| 情况 | Phase 3 处理 | 不得做什么 |
|---|---|---|
| 来源状态为 `未获得` | 放入待补证据，说明缺口和需要补什么 | 不得写成已获得证据 |
| 来源状态为 `不适用` | 简要说明为什么本轮不需要该来源 | 不得为了凑齐三源而强行补检索 |
| 字段缺失 | 将缺少的 PMID、DOI、年份、期刊、URL、摘要等标注为 `未提供` | 不得凭记忆或常识补全 |
| 弱来源或普通网页 | 保留 Phase 2 的弱来源标注，只作为线索 | 不得升级为正式医学证据 |
| 用户材料 | 标注为用户提供材料，并说明覆盖范围 | 不得声称代表全领域系统检索结果 |

若来源没有实际返回结果，不要编造代表性文献、数据集、试验编号、指南结论、引用次数或 landmark 地位。

## 文献与资源分布

Phase 3 可以做描述性概述，但不要做创新性判断。优先从下列维度整理：

| 维度 | 需要报告 |
|---|---|
| 年份趋势 | 早期 / 稳定 / 上升 / 近年爆发 / 不清楚 |
| 研究类型 | 综述、Meta、临床、组学、细胞、动物、试验、方法学、网页资源 |
| 疾病和模型覆盖 | 疾病阶段、组织来源、细胞类型、动物/细胞模型、公共数据集 |
| 方法 | qPCR、IHC、western blot、测序、单细胞、蛋白组、生信、功能实验、算法模型 |
| 终点 | 诊断、预后、生存、治疗反应、分子表型、数据资源可用性 |
| 机制主题 | 通路、细胞状态、生物过程、标志物、靶点、治疗背景 |

如果材料不足以判断某个维度，写 `当前材料不足以判断`。

## 来源贡献与边界

不同来源只能支持不同层级的结论。Phase 3 需要说明贡献，也要说明边界。

| 来源类型 | 可以支持 | 不能支持 |
|---|---|---|
| PubMed 核心医学文献 | 医学背景、机制线索、临床相关性、代表性研究、已有研究密度 | 不自动证明因果、疗效、创新性或可发表性 |
| arXiv 方法学与预印本 | 方法路线、算法线索、多组学或 AI 可迁移思路 | 不能替代医学证据，不能单独支持临床或机制结论 |
| Web/Tavily 网页与数据库核验 | 数据集、试验注册、指南、官方页面、数据库入口、论文线索补查 | 不能替代正式文献证据，弱来源只能作为线索 |
| 用户提供材料 | 本轮用户实际语料中的主题、结论、实验结果或限制 | 不能代表全领域系统检索结果 |

## Phase 3 输出顺序

除非用户要求极简回答，Phase 3 按以下顺序输出：

1. 材料范围与来源状态。
2. 文献与资源分布。
3. 来源贡献与边界。

各部分字段应以上文对应章节为准，不在此处重复表格。
