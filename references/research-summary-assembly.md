# 科研总结报告装配模板

本文件用于 Phase 7。它接收 Phase 1-6 已经生成的报告片段和阶段产物，将标准化问题、标准化证据包、文献与资源证据概述、创新点证据地图、研究空白审计结果、候选假说和验证路径装配成一份可保存、可下载、可追溯的 Markdown 总结报告。

Phase 7 是报告装配阶段，不是新的分析阶段。它不额外思考，不重新检索，不重新归纳证据，不重新提炼候选方向，不重新审计研究空白，不重新生成假说，也不补充新的判断。若前序阶段没有提供某个报告片段或字段，Phase 7 只能标注 `未提供` 或 `该阶段片段缺失`。

除非用户要求极简回答，一般情况下Phase 7 默认输出 Markdown 报告正文；建议文件名为 `research-summary-[topic-slug]-[YYYYMMDD].md`。

## 输入材料

Phase 7 只汇总和装配已经形成的阶段产物，不回头重新检索、重新提炼候选方向或重新生成假说。

| 输入 | 用途 |
|---|---|
| Phase 1 标准化问题 | 确认原始问题、研究边界和待补信息 |
| Phase 2 标准化证据包 | 确认来源范围、返回记录和用户材料状态 |
| Phase 3 文献与资源证据概述 | 确认证据分布、来源贡献和待补证据 |
| Phase 4 创新点证据地图 | 确认候选方向、主类型和主要证据依据 |
| Phase 5 候选方向审计表 | 确认保留、降级、剔除及主要风险 |
| Phase 6 候选假说与验证路径 | 整理可检验假说、验证路径和预期产出 |

## 报告输出顺序

Phase 7 按以下顺序输出 Markdown 总结报告：

1. 报告元信息。
2. 研究问题与边界。
3. 证据来源与材料范围。
4. 文献与资源证据概述。
5. 创新点证据地图。
6. 研究空白审计结果。
7. 候选假说与验证路径。
8. 主要风险与待补证据。
9. 后续使用建议。

### 验证路径摘要

使用 Phase 6 已给出的验证路径，基于 Phase 2-6 已有证据和用户可用资源选择下一步版本，不在 Phase 7 新增检索任务或重写完整课题方案。

| 版本 | 适用条件 | 核心工作 | 产出 |
|---|---|---|---|
| Lite | 只有文献、已有证据包或公共数据线索 | 基于已有证据包和可获得公共数据，规划表达、通路、生存、相关性或初步证据图分析 | protocol seed、初步证据图或待补证据清单 |
| Standard | 有临床样本或基础实验条件 | 本地样本验证、qPCR/IHC/ELISA/western blot、流式、简单功能实验 | 原创研究路径或小规模验证结果 |
| Publication+ | 有更强协作和平台资源 | 独立队列、多组学、单细胞/空间、动物或 rescue、外部验证 | 更强机制或转化研究路径 |

可以说明不同验证版本各自适合什么资源条件，但不强制选择唯一版本。

## 八、主要风险与待补证据

汇总 Phase 3-6 中仍会影响后续使用、讨论或方案设计的风险，不重复所有阶段细节。

| 风险或缺口 | 来自阶段 | 影响 | 建议处理 |
|---|---|---|---|

常见来源包括：证据来源未获得、证据层级较低、结论冲突、因果过度、转化过度、数据/样本不可得、验证负担过重。

## 九、后续使用建议

用 2-5 条说明这份报告后续可以如何使用，例如：

- 用于和导师讨论研究切入点；
- 用于补充检索或整理 evidence packet；
- 用于选择 Lite / Standard / Publication+ 验证路径；
- 用于后续另行生成研究方案、开题草案或实验设计。
