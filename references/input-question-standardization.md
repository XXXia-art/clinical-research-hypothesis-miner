# 用户输入与研究问题标准化

本文件用于 Phase 1。目标不是直接回答科研结论，而是把用户的原始输入转成后续证据路由、文献概述、创新点地图、假说生成和科研决策可使用的标准化问题包。

## Phase 1 目标

必须完成四件事：

1. 识别用户实际提供了什么材料。
2. 判断研究问题是否足够明确，缺什么关键要素。
3. 将模糊中文想法改写为可检索、可审计、可验证的科研问题。
4. 输出实体表、同义词、歧义边界和待补信息，为 Phase 2 检索路由做准备。

不要在 Phase 1 就判断“已证实”“无人研究”“一定创新”或“可以投稿”。这些判断必须等待后续证据包。

## 输入材料分类

先判断输入属于哪一类，可多选：

| 类型 | 识别线索 | Phase 1 处理 |
|---|---|---|
| 初始 idea | 只有疾病、机制、基因、药物、临床现象或一句话方向 | 标准化问题，列缺失要素，整理核心实体 |
| 关键词列表 | 用户给出多个疾病、基因、通路、药物或终点 | 整理实体表，识别同义词和冲突符号 |
| 已有论文材料 | 标题、摘要、PMID、DOI、文献列表、用户整理的笔记 | 标注为 `用户提供`，不得假装重新检索 |
| 工具返回材料 | academic-research/PubMed、arXiv、Web/Tavily 的结果 | 保留来源状态，进入后续 evidence packet |
| 实验或数据结果 | 差异基因、测序结果、临床样本结果、动物/细胞实验结果 | 标注证据类型和外推边界 |

如果用户材料很少，仍然可以继续输出标准化问题和核心实体整理；但必须说明当前证据状态是 `计划中` 或 `用户提供`，不是 `已检索`。

## 最小必要要素

优先抽取以下要素：

| 要素 | 说明 | 缺失时处理 |
|---|---|---|
| 疾病/表型 | 疾病、综合征、病理状态、并发症、分期或亚型 | 若过宽，提示需要收窄到疾病亚型、阶段或场景 |
| 核心实体 | 基因、蛋白、细胞亚群、药物、代谢物、通路、机制 | 未给出具体实体时标注 `待指定`，不得自行指定唯一候选 |
| 研究用途 | 机制解释、诊断、预后、治疗反应、靶点验证、耐药机制 | 若用途混杂，拆成多个问题 |
| 研究对象 | 患者、组织、细胞、动物模型、公共数据集、临床样本 | 若物种或样本来源不明，标注外推风险 |
| 终点 | 发病、进展、死亡、治疗应答、标志物性能、通路激活 | 若终点不可测量，改写为可测终点 |

## 追问与降级规则

只有在缺失信息会导致问题完全不可标准化时才追问。例如用户说“某个炎症基因”但没有给出基因名，且任务要求判断该基因是否能做靶点，此时应先要求用户补充基因符号。

如果仍能形成初步问题，则不要停下等待。应采用降级输出：

- 未给基因：标注 `候选基因待指定`，只整理疾病 + 机制层面的核心实体。
- 未给疾病亚型：保留宽问题，同时建议 2-3 个可收窄方向。
- 未给终点：把“有没有用”拆成诊断、预后、治疗反应、机制验证等不同终点。
- 未给资源约束：默认不承诺动物、单细胞、多组学或临床队列可执行，只列为可选验证层级。

## 核心实体与边界表

这是 Phase 1 输出模板中的核心子模块，用于承接“标准化研究问题”，并为后续证据路由、检索式构建和证据边界判断提供统一实体。不要在最终回答中额外生成另一张“标准化输出表”；只需要在 Phase 1 模板的“核心实体与边界”部分使用这张表。

| 项目 | 标准化结果 | 同义词/检索词 | 排除范围或歧义 |
|---|---|---|---|
| 疾病/表型 | 英文标准疾病名 + 中文名 | 同义词、缩写、相关疾病 | 排除疾病、分期限制、异质性亚型 |
| 机制/通路 | 标准生物过程或通路名 | 通路同义词、上游/下游机制词 | 容易引入噪声的宽泛词 |
| 基因/蛋白/药物/代谢物 | 已知官方符号或标准名 | 别名、全称、常见旧名 | 未解决或冲突的符号 |
| 细胞/组织/模型 | 人群队列、组织、细胞类型、动物模型 | 模型相关词 | 物种不匹配、模型不等于临床 |
| 研究用途/终点 | 诊断、预后、治疗反应、机制、靶点验证 | 终点同义词 | 模糊或不可测量终点 |
| 资源约束 | 用户已说明的数据、样本和实验条件 | 可用资源关键词 | 不可假定存在的资源 |

未解决项目标注为 `待核验`、`待指定` 或 `用户未提供`。不得编造基因 ID、登录号、MeSH ID、试验 ID、数据集编号或数据库记录。

## 问题改写

必要时输出三个版本：

| 版本 | 作用 | 写法要求 |
|---|---|---|
| 临床中文问题 | 让临床医生能直接理解 | 保留真实临床现象，避免过早抽象成机制口号 |
| 科研问题 | 供后续证据综合、创新点地图和假说生成使用 | 明确疾病、实体、终点、研究对象和边界 |
| 英文检索问题 | 供 academic-research/PubMed、arXiv 和 Web/Tavily 后续路由使用 | 使用标准英文医学术语、基因/蛋白符号、通路名和可检索终点 |


## 检索词分层

按层构建检索词。先宽后窄，不要一开始把检索式写得过窄，导致领域全貌被遮蔽。

| 层级 | 内容 | 示例 |
|---|---|---|
| L1 疾病词 | 英文标准疾病名、缩写、相关综合征、分期 | sepsis, septic shock, acute kidney injury, SA-AKI |
| L2 实体词 | 官方基因/蛋白符号、全称、常见别名、药物名、代谢物 | GPX4, SLC7A11, CD48, sphingosine-1-phosphate |
| L3 机制词 | 生物过程、通路、细胞状态 | ferroptosis, macrophage polarization, immune exhaustion, metabolic reprogramming |
| L4 用途词 | 研究目标和应用场景 | biomarker, prognosis, diagnosis, therapeutic target, resistance, mechanism |
| L5 证据类型词 | 研究设计、数据类型、模型 | transcriptomics, proteomics, single-cell, cohort, animal model, clinical trial |
| L6 约束词 | 年份、物种、组织、疾病亚型、治疗类型 | human, mouse, renal tubular epithelial cell, anti-PD-1 |

## 歧义检查

下结论前标注以下风险：

- 同一缩写有多个生物医学含义。
- 基因符号与英文普通词、旧别名或家族成员重叠。
- 中文药物名、代谢物名或细胞亚群名存在多个英文译名。
- 疾病名包含异质性亚型，如不同分期、病因、治疗线数或组织来源。
- 生物标志物问题没有定义诊断、预后、疗效预测或机制终点。
- 治疗靶点问题只有相关性证据，缺少因果性、可药性和安全性要素。
- 多组学问题没有说明样本匹配、批次、临床变量和验证队列。
- 公共数据问题没有确认数据集真实存在、样本量、平台和可下载性。
- 细胞或动物实验结果被用户表述为临床结论。

如果歧义会影响结论，应收窄推荐，而不是猜测。

## 不可做事项

- 不得把标准化阶段写成文献综述。
- 不得凭记忆补 PMID、DOI、MeSH ID、GEO/TCGA 登录号或临床试验编号。
- 不得把“没有看到用户提供材料”写成“数据库没有相关研究”。
- 不得自动把“相关基因”升级为“临床治疗靶点”。
- 不得把“加机器学习”“加单细胞”“加多组学”直接视为创新。
