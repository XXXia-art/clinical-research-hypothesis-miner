# 三源信息汇总与搜索结果概述

当已经获得用户提供语料，或已经从 [证据来源路由](evidence-source-routing.md) 得到 PubMed、arXiv、Web/Tavily 原始来源包后，使用本参考文件。

本文件只回答：**如何把三源信息汇总成文献概述、证据边界和待补证据清单。**

不要在本文件中设计检索式、决定工具调用或抓取新结果；这些任务交给 `evidence-source-routing.md`。

汇总时必须保留三源的来源性质差异：

- PubMed 原始来源包来自本 Skill 直接执行或规划的 PubMed 检索，是核心医学证据主干的输入。
- arXiv 原始来源包来自本 Skill 的方法学/预印本检索或检索计划，只能支持方法线索。
- Web 原始来源包来自本 Skill 的网页/官方来源核验或核验计划，只能支持数据源、试验、指南、论文线索和背景边界。

## 输入类型

接受并标准化：

- PubMed 原始来源包：PubMed 检索返回的标题、摘要、年份、期刊、PMID/DOI、检索式、缺失字段和来源边界。
- arXiv 原始来源包：本 Skill 获取或规划的 AI、统计模型、多组学算法、计算框架或预印本方法线索。
- Web 原始来源包：本 Skill 获取或规划的官方页面、数据集、临床试验、指南、数据库入口、论文线索补查或背景核验结果。
- 用户提供语料：标题和摘要、文献管理器导出文本、用户已筛选论文列表、混合来源笔记。

如果元数据缺失，标注为缺失。不得凭记忆补 DOI、PMID、引用次数、期刊或年份。

## 汇总前检查

在概述前，先检查每个来源的执行状态：

| 来源 | 状态处理 |
|---|---|
| 已检索 | 可以作为本轮实际材料概述，但仍需说明检索范围和限制 |
| 用户提供 | 只能代表用户语料，不能声称是系统检索全貌 |
| 计划中 | 只能列为待补证据，不能写入已知证据结论 |
| 不适用 | 简要说明为什么不需要该来源 |

若某来源没有实际返回结果，不要编造代表性文献、数据集、试验编号或指南结论。

如果 PubMed evidence packet 缺失，应写成“尚未执行/尚未获得 PubMed 检索返回”，而不是自行替代完成 PubMed 综述。

## 三源汇总步骤

1. **按来源建账。** 将材料分为核心医学文献、方法学与预印本补充、网页与数据库核验、用户提供材料。
2. **标准化记录。** 在信息存在时提取标题、年份、期刊/来源、检索来源、摘要片段、实体、疾病、方法、模型和终点。
3. **谨慎去重。** 只合并明显重复标题；不要因为主题相似就合并记录。
4. **分类研究类型。** 综述/Meta、人群临床、公共组学、细胞实验、动物实验、临床试验、数据库/方法学论文、网页背景资料。
5. **提取跨源主题。** 机制、细胞类型、通路、生物标志物、靶点、模型、治疗背景、终点、算法或数据资源。
6. **区分证据层级。** PubMed 医学文献优先支撑医学/机制结论；arXiv 只支撑方法学线索；Web/Tavily 只支撑页面核验、数据源线索、试验或指南入口。
7. **评估证据边界。** 说明三源合起来能支持什么，不能支持什么。
8. **识别待补证据。** 指出缺失来源、缺失模型、缺失临床终点、缺失验证层级或需要重新检索的方向。

## 必要概述部分

### 材料范围与来源状态

- 语料实际覆盖的主题。
- 涉及的来源。
- 已知时间范围。
- 每个来源的执行状态：已检索、用户提供、计划中、不适用。
- 结果是 top hits、用户筛选结果、系统检索结果，还是待检索计划。

### 三源贡献

| 来源类型 | 需要概述 | 不得越界 |
|---|---|---|
| 核心医学文献 | PubMed 检索返回的医学/机制/临床相关性证据、代表性研究、证据强弱 | 不得补入 PubMed 未返回的文献记录 |
| 方法学与预印本补充 | arXiv 中 AI、统计、多组学、计算框架的可借鉴方法 | 不得替代医学证据或临床结论 |
| 网页与数据库核验 | Web/Tavily 中数据集、临床试验、指南、官方页面和数据库入口 | 不得把网页片段当作正式文献证据 |
| 用户提供材料 | 用户语料中的主题、机制、结论和限制 | 不得声称代表全领域检索结果 |

### 文献分布

| 维度 | 需要报告 |
|---|---|
| 年份趋势 | 早期/稳定/上升/近年爆发/不清楚 |
| 研究类型 | 综述、临床、组学、细胞、动物、试验、方法学 |
| 疾病/模型覆盖 | 人群疾病阶段、组织、动物/细胞模型 |
| 方法 | qPCR、IHC、western blot、测序、单细胞、蛋白组、生信、功能实验 |
| 终点 | 诊断、预后、生存、治疗反应、分子表型 |

### 代表性文献

只从用户提供或实际检索到的材料中列出代表性记录：

| 文献 | 来源 | 年份 | 重要性 | 限制 |
|---|---|---:|---|---|

如果来源未提供年份或期刊，写 `未提供`。

### 待补证据

把未实际执行或证据不足的来源放到待补证据清单，而不是混入已知结论：

| 待补来源 | 需要补什么 | 为什么重要 | 推荐状态 |
|---|---|---|---|
| PubMed | 核心医学证据、系统综述、机制/临床研究 | 决定医学结论是否站得住 | 必补/可选/不适用 |
| arXiv | 方法学、算法、多组学流程、预印本线索 | 决定方法是否新、是否可迁移 | 必补/可选/不适用 |
| Web/Tavily | 数据集、临床试验、指南、官方页面 | 决定资源和转化边界是否真实 | 必补/可选/不适用 |

### 语料能支持什么

使用克制表述：

- “支持相关性线索”。
- “提示机制可能参与”。
- “存在功能验证基础”。
- “已有转化/临床探索”。
- “目前仅能作为假说”。

### 语料不能支持什么

明确列出不能过度宣称的内容：

- 临床疗效。
- 诊断落地应用。
- 从表达相关性直接推出因果。
- 靶点成药性。
- 外推到其他人群或疾病阶段。

## 输出风格

概述要简洁但结构清楚。除非用户要求，不要写成长篇逐篇注释书目。
