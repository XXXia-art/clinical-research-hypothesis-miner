# 假说与验证设计

当搜索结果概述和创新点证据地图已经识别出候选机会后，使用本参考文件。

## 假说要求

每个假说都必须能追溯到证据地图：

| 字段 | 要求 |
|---|---|
| 假说 | 一句可检验表述 |
| 证据锚点 | 语料信号、PubMed 结果、arXiv 方法线索、网页/数据库核验结果，或清楚标注的推断 |
| 机制链 | 触发因素 -> 分子事件 -> 细胞/组织表型 -> 可测量终点 |
| 关键读出 | 基因、蛋白、通路、细胞状态、临床终点或组学特征 |
| 最小验证 | 能检验核心主张的最小数据集或实验 |
| 失败解释 | 什么结果会削弱、反驳或重定向该假说 |

避免只说“X 与 Y 相关”的假说。必须包含方向、机制、场景和可测读出。

## 假说生成句式

使用如下结构：

```text
In [disease/model/context], [entity/pathway/cell state] may drive or mark [phenotype/outcome] through [mechanism], and this can be tested by [minimal validation].
```

中文回答时可翻译为：

```text
在[疾病/模型/场景]中，[实体/通路/细胞状态]可能通过[机制]驱动或标记[表型/结局]，可通过[最小验证]进行检验。
```

## 验证阶梯

### Lite

适用于用户只有文献和公共数据的情况。

- 重新运行 PubMed 检索并收集核心论文。
- 按需使用 arXiv 查找 AI、计算方法、多组学算法或预印本方法线索。
- 使用 Web/Tavily 查找公共数据集、官方数据库页面、临床试验注册或指南页面。
- 识别公共数据集或官方数据集页面。
- 如果可获得数据，规划表达、通路、生存或相关性分析。
- 产出初步证据图或研究 protocol。

### Standard

适用于用户有临床样本或基础实验条件的情况。

- 在本地样本中验证表达或通路信号。
- 按需使用 qPCR、IHC、ELISA、western blot、流式或面向 panel。
- 如果主张涉及机制，加入简单功能细胞实验。
- 保证终点与假说一致。

### Publication+

适用于用户有协作资源和更强平台条件的情况。

- 只有当单细胞、空间组学或多组学能回答明确机制/细胞状态问题时才加入。
- 只有在因果验证必要时才加入动物模型和 rescue 实验。
- 对 biomarker 主张加入独立队列或外部验证。
- 只有在机制和安全边界清楚后，才考虑转化延伸。

## 可行性匹配

如果用户未说明资源，默认按现实医院科研条件估计：

- 文献和 PubMed 检索可用。
- arXiv 可用于方法学和预印本补充。
- Web/Tavily 可用于官方页面、数据集、临床试验和背景核验。
- 公共数据集可能可用，但需要核验。
- 临床样本需要伦理审批。
- 昂贵的单细胞、空间组学、蛋白组或动物实验应作为升级路径，而不是默认要求。

## 最终假说选择

最多推荐三个假说：

- **主假说：** 证据和可行性最强。
- **备选假说：** 风险更低或更容易验证。
- **进阶假说：** 创新性更高，但资源负担更大。

如果当前证据不足以形成可靠假说，输出 `Redesign`，并说明缺少哪些证据。
