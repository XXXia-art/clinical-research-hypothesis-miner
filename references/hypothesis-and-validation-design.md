# 假说与验证设计

本文件用于 Phase 6。它直接接收 Phase 5 候选方向审计表中的 `保留` 或 `降级` 方向，并回溯 Phase 4 创新点证据地图中的主要证据依据，将审计后的研究空白转化为 1-3 个可检验假说和分层验证路径。

Phase 6 不重新检索，不重新绘制证据地图，不重新判断伪空白，也不做最终 Start/Narrow/Redesign/Stop 决策。它只回答：

1. 哪些审计后方向可以形成可检验假说。
2. 每个假说如何回到 Phase 5 审计依据和 Phase 4 证据地图。
3. 在 Lite、Standard、Publication+ 三种资源条件下，最小验证路径分别是什么。
4. 如果证据不足以生成假说，需要回退到补证据、收窄或重构。

## 输入材料

Phase 6 只接收 Phase 5 已审计后的候选方向，并按需回溯 Phase 4 证据地图。不得接收未经 Phase 5 审计的 Phase 4 方向。

| 输入 | 用途 |
|---|---|
| Phase 5 候选方向 | 作为假说生成对象 |
| Phase 5 审计结果 | 只允许 `保留` 和 `降级` 进入本阶段 |
| Phase 5 审计依据 | 决定假说表述强度和边界 |
| Phase 5 主要风险 | 决定验证路径必须覆盖哪些风险 |
| Phase 5 待补证据或收窄建议 | 决定是否先收窄或只生成低强度假说 |
| Phase 4 主要证据依据 | 作为假说的证据地图来源 |

## 假说表达类型

根据候选研究空白选择最合适的假说类型，不强制所有假说都写成机制因果句。

| 假说类型 | 适用场景 | 表达重点 |
|---|---|---|
| 机制假说 | 需要解释疾病、表型或干预反应的生物学过程 | 实体 -> 机制事件 -> 细胞/组织表型 -> 可测终点 |
| 标志物假说 | 关注诊断、预后、疗效预测或风险分层 | 实体或特征组合能否区分人群、结局或治疗反应 |
| 分层假说 | 关注异质性人群、亚型、疾病阶段或治疗情境 | 哪类人群/亚型中信号更强，边界在哪里 |
| 网络或通路假说 | 关注多基因、多通路、细胞状态或调控模块 | 网络结构、关键节点、模块变化和可验证读出 |
| 方法学假说 | 关注 AI、算法、多组学整合或计算框架 | 方法能否改善识别、解释、预测或验证效率 |

可使用下列表达模板，但不要机械套用：

- **机制类：** 在[疾病/模型/场景]中，[实体/通路/细胞状态]可能通过[机制事件]影响[表型/结局]，可通过[关键读出]检验。
- **标志物类：** 在[目标人群/疾病阶段]中，[实体/特征组合]可能标记或预测[结局/疗效/风险]，可通过[队列或数据集]检验其区分能力和稳定性。
- **分层类：** 在[异质性人群]中，[亚型/状态/暴露]可能定义一个[高风险/高反应/特殊机制]亚群，可通过[分层分析或交互检验]验证。
- **网络类：** 在[疾病或数据场景]中，[通路/细胞状态/调控模块]可能形成与[表型/结局]相关的网络结构，可通过[模块稳定性、关键节点或正交数据]检验。
- **方法类：** 在[数据或任务场景]中，[方法/模型]可能比[基线方法]更好地识别或解释[目标信号]，可通过[外部验证、消融实验或可解释性检查]检验。

## 空白到假说转化

Phase 6 只处理 Phase 5 审计结果为 `保留` 或 `降级` 的候选方向；`剔除` 方向不得进入假说生成。Phase 5 决定哪些方向可以进入本阶段，Phase 4 提供这些方向的证据结构和主要证据依据。

用下表把审计后空白转成假说对象：

| 审计后空白 | Phase 5 审计结果 | Phase 4 证据地图依据 | 假说类型 | 核心输入 | 可形成的假说方向 |
|---|---|---|---|---|---|

字段要求：

- **审计后空白：** 来自 Phase 5 候选方向审计表，不新增未审计方向。
- **Phase 5 审计结果：** `保留` 可优先转化为假说；`降级` 只能转化为边界更窄、证据表述更克制的假说。
- **Phase 4 证据地图依据：** 回填创新点证据地图中的候选方向、主类型、主要证据依据和候选机会说明；不得写成脱离证据地图的新依据。
- **假说类型：** 从机制、标志物、分层、网络/通路、方法学中选择最合适的一类。
- **核心输入：** 写明形成该假说至少需要哪些已有证据、公共数据、样本、模型或方法条件。
- **可形成的假说方向：** 只写假说雏形，不展开完整验证方案。

## 验证阶梯

### Lite

适用于用户只有文献和公共数据的情况。

- 使用 Phase 2-5 已获得的文献、证据包、证据地图和审计结果。
- 识别已经出现的公共数据集、官方数据库页面、临床试验注册或指南页面。
- 如果现有材料显示数据可获得，规划表达、通路、生存或相关性分析。
- 若关键证据缺失，标注为待补证据，不在本阶段重新检索。
- 产出初步证据图或研究 protocol。

### Standard

适用于用户有临床样本或基础实验条件的情况。

- 在本地样本中验证表达或通路信号。
- 按需使用 qPCR、IHC、ELISA、western blot、流式或面向 panel。
- 如果主张涉及机制，加入简单功能细胞实验。
- 保证终点与假说一致。

### Publication+

适用于用户有协作资源和更强平台条件的情况。

- 只有当单细胞、空间组学或多组学能回答明确机制/细胞状态问题时才加入。
- 只有在因果验证必要时才加入动物模型和 rescue 实验。
- 对 biomarker 主张加入独立队列或外部验证。
- 只有在机制和安全边界清楚后，才考虑转化延伸。

## 输出

候选假说与验证路径可以使用卡片式结构，便于用户阅读：

```md
### 假说 H1：候选假说名称

**来源候选方向：** 对应 Phase 5 审计后的候选方向。  
**假说类型：** 机制 / 标志物 / 分层 / 网络或通路 / 方法学。  
**假说内容：** 用一段话说明可检验的科学假说。  
**主要风险：** 引用 Phase 5 的主要风险，避免过度宣称。

**验证路径（Lite / Standard / Publication+ 中选择适合的一档）：**

| 步骤 | 内容 | 数据库/工具/材料 |
|---|---|---|
|  |  |  |

**预期产出：** 只写该验证路径可能产生的直接结果，不写最终启动决策。
```
