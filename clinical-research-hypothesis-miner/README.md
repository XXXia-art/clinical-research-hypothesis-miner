# 临床科研文献到假说挖掘器

这是一个面向临床医生、医学生和基础医学研究者的科研决策 Skill。它的核心目标不是替用户凭空生成 idea，而是从 PubMed、用户提供的 Google Scholar 结果、论文列表、摘要和网页核验线索中整理证据，再推导创新点、研究空白、可检验假说和可执行研究方案。

## 核心能力

- 将中文临床问题、疾病机制、靶点、通路、药物或终点标准化为可检索科研问题。
- 以 PubMed 作为核心医学文献入口，结合网页核验和用户提供的 Google Scholar 结果进行广泛召回与引用网络判断。
- 对检索到或用户提供的文献进行概述，说明研究对象、模型、方法、机制主题、证据层级和证据边界。
- 构建创新点证据地图，区分证据密集区、薄弱证据区、冲突证据区、可转化空白和伪创新点。
- 从证据地图中生成 1-3 个可追溯、可检验、可落地的科研假说。
- 提供 Lite、Standard、Publication+ 三档验证路径，并输出 Start / Narrow / Redesign / Stop 科研决策。

## 典型输入

- 一个初步科研 idea，例如“铁死亡是否参与脓毒症相关急性肾损伤”。
- 一组关键词、基因、蛋白、通路、疾病或药物。
- PubMed 检索结果、Google Scholar 检索结果或用户整理的论文列表。
- 若干篇论文的标题、摘要、结论或 PMID/DOI。
- 一个需要判断是否值得继续投入的临床科研方向。

## 典型输出

- 标准化科研问题与英文检索式。
- 文献来源与证据范围概述。
- 代表性文献和证据边界。
- 创新点证据地图。
- 被剔除的伪空白与保留的真实研究空白。
- 可检验假说与失败解释。
- 可执行研究方案。
- Start / Narrow / Redesign / Stop 决策建议。

## 工具边界

本 Skill 可以使用平台提供的 PubMed 和 Web 检索类 MCP。Google Scholar 当前按用户提供结果、手动检索计划或未来平台 MCP 处理，不会伪装成真实已调用的 Scholar 工具。

如果 draw.io MCP 可用，它只用于把证据地图、机制链或验证路径画成图，不负责检索、查证或判断创新性。draw.io 不可用时，可以输出 Mermaid 或 draw.io XML 草稿供用户手动导入。

## 安全与严谨性

- 不编造论文、作者、期刊、年份、DOI、PMID、基因 ID、数据库记录或试验编号。
- 不把表达相关性直接说成因果关系。
- 不把细胞或动物证据直接上升为临床疗效。
- 不把“加单细胞”“加多组学”“扩大样本量”“用机器学习”当作天然创新。
- 不提供患者个体化诊断或治疗建议，所有输出限定为科研规划。

## 一句话定位

这个 Skill 帮助用户回答：这个科研方向有没有真实创新空间，证据支持到哪一步，下一步应该怎么做，以及是否值得继续投入。
